Rôles & Permissions

L'application gère 7 rôles distincts. Chaque rôle détermine les fonctionnalités accessibles. Votre rôle est visible sous forme de badge coloré dans la barre de navigation.

👤 Présentation des rôles

👑
super_admin Super Admin

Accès complet à toutes les fonctionnalités de l'application. Peut gérer les utilisateurs, administrer les checklists, les sections CPIA, les paramètres globaux, créer/modifier/archiver tous les projets et accéder aux revues Facility Manager.

🏢
facility_manager Facility Manager

Peut créer et modifier des projets, gérer les inspections QA, accéder aux revues FM, modifier les paramètres de l'application. Ne peut pas gérer les utilisateurs ni administrer les checklists/CPIA.

🔬
qa_manager QA Manager

Accès au tableau de bord QA, à la planification et au suivi des inspections, aux checklists d'inspection et aux constatations. Peut télécharger les documents QA. Ne peut pas créer de projets.

🧪
study_director Study Director

Voit uniquement ses propres projets (là où il est study director actif). Peut gérer le protocole, compléter le CPIA, télécharger les documents de ses projets. Ne peut pas créer de projets.

📁
project_manager Project Manager

Voit uniquement les projets dont il fait partie (personnel clé). Peut gérer le protocole et compléter le CPIA. Ne peut pas créer de projets.

🗄️
archivist Archivist

Accès à la phase d'archivage des projets. Peut soumettre et gérer les documents d'archivage.

👁️
read_only Read Only

Consultation uniquement. Peut voir tous les projets et les données mais ne peut effectuer aucune modification.
