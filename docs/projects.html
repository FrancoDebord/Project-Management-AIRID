🗂️ Créer & Gérer un Projet (Étude)

L'application Project Tracking Sheet suit chaque étude à travers un processus en 8 étapes structurées (6 pour les études Non-GLP). Cette page explique chaque étape, ce qu'on y fait, qui peut y intervenir et comment progresser.

Créer un nouveau projet

Les utilisateurs habilités (Super Admin, Facility Manager, Study Director) peuvent créer une étude depuis le Tableau de bord en cliquant sur "Nouveau projet".

Champs requis à la création :
  • Code de l'étude — identifiant unique (ex. : 1201)
  • Titre de l'étude — intitulé complet
  • Type(s) d'étude — Laboratory Bioassays, Experimental Hut Trials, Community Trials, Resistance Bioassays (sélection multiple)
  • GLP ? — Oui ou Non. Ce choix détermine si les étapes Planning (4), QA (6) et Archivage (8) sont actives
  • Study Director — optionnel à la création, peut être renseigné à l'étape 1

Une fois créé, vous êtes redirigé automatiquement vers la fiche de gestion du projet.

Statuts d'une étude

Chaque étude possède un statut global qui détermine si elle est modifiable :

StatutSignificationModifications
Non démarréeL'étude est enregistrée mais pas encore lancée❌ Bloquées
SuspendueL'étude est temporairement arrêtée❌ Bloquées
En coursL'étude est active et modifiable✅ Autorisées
ComplétéeToutes les phases sont finalisées✅ Autorisées
ArchivéeL'étude est archivée et verrouillée❌ Bloquées
Quand une étude est Non démarrée ou Suspendue, une bannière d'avertissement s'affiche en haut de la fiche et tous les champs sont grisés. Seuls le Facility Manager et le Super Admin peuvent changer le statut via le bouton "Lancer / Reprendre l'étude" dans la bannière.

Étape 1 — Study Creation

Qui : Facility Manager, Study Director, Project Manager, Super Admin

Cette étape recueille toutes les informations fondamentales avant que l'étude ne puisse avancer.

📌 Project Basic Information

Informations générales de l'étude : code, titre, nature, protocole, système test, phase courante. Ces données peuvent être modifiées via le bouton "Edit Project Info".

📌 Study Director Appointment Form

Désignation officielle du Study Director (SD) : nom, qualification, date de désignation, fichier d'appointment joint. Le Project Manager peut également y être renseigné. Ce formulaire génère une notification au SD concerné.

  • Une seule nomination active à la fois (la précédente passe en historique si remplacement)
  • Le fichier d'appointment (PDF/Word) peut être téléchargé depuis cette section

📌 Study Director Replacement History

Historique automatique de tous les Study Directors ayant occupé le rôle sur cette étude. Chaque ligne affiche le nom, la date de nomination, la date de remplacement et le motif. Ce tableau est en lecture seule.

📌 Other Basic Documents

Documents de base complémentaires (protocoles préliminaires, lettres de mission, etc.) que vous pouvez uploader avant de démarrer l'étude. Chaque document renseigné contient : titre, description, fichier joint, date d'upload.

📌 Key Personnel

Liste du personnel clé affecté à l'étude (autres que le SD et PM). Géré en ligne : on sélectionne une personne dans la liste du personnel AIRID et on clique "Add". Un clic sur 🗑 retire la personne de l'équipe. Chaque ajout ou retrait déclenche une notification à la personne concernée.

📌 Progression

Carte de progression globale avec un indicateur en % et 6 métriques de suivi (activités, documents Protocol Dev, inspections QA, NC findings, rapport, archivage) présentées sous forme de chips colorées.

Complétion de l'étape 1 : Cette étape est marquée complète lorsque le Study Director Appointment Form est renseigné et qu'au moins un document ou une activité clé est enregistrée.

Étape 2 — Protocol Details

Qui : Facility Manager, Study Director, Project Manager, Super Admin

Cette étape centralise toutes les informations détaillées du protocole d'étude :

  • Type(s) d'étude — avec possibilité d'en ajouter ou retirer
  • Nature du projet (terrain, labo, etc.)
  • Produit(s) testé(s) — types de produits évalués
  • Tests de laboratoire concernés
  • Durée estimée — date de début et de fin prévues
  • Description générale de l'étude

Ces informations alimentent le Gantt, le Master Schedule et les rapports PDF.

Étape 3 — Protocol Development

Qui : Facility Manager, Study Director, Project Manager, Super Admin

Suivi des activités de développement du protocole. Cette étape liste les tâches standards du développement (définies par le système) et permet de :

  • Marquer chaque activité comme complétée
  • Joindre un document (preuve de complétion)
  • Ajouter des entrées manuelles si besoin

La progression est calculée automatiquement en pourcentage d'activités complètes.

Étape 4 — Planning Phase (GLP uniquement)

Qui : Facility Manager, QA Manager, Super Admin

Cette étape n'est disponible que pour les études GLP. Elle couvre :

📌 Réunion d'initiation (Study Initiation Meeting)

Planification et enregistrement de la réunion d'initiation entre le Study Director, le QA Manager et les parties prenantes clés. Contient : date, lieu, lien de visio, ordre du jour, liste des participants.

  • Cette réunion est un prérequis pour accéder à la CPIA (étape 4bis)
  • Un email de notification est envoyé aux participants à la planification

📌 Activités planifiées (Gantt)

Liste de toutes les activités de l'étude avec dates estimées et réelles, statut, responsable. On peut :

  • Ajouter / modifier / supprimer des activités
  • Marquer une activité comme Phase Critique (obligatoire pour la CPIA)
  • Exécuter une activité (enregistrer la date réelle de réalisation)

📌 CPIA — Critical Phase Impact Assessment

Évaluation de l'impact des phases critiques par le QA Manager. Accessible depuis un bouton dédié dans la fiche projet. → Voir la documentation CPIA

Étape 5 — Experimental Phase

Qui : Facility Manager, Study Director, Project Manager, Super Admin

Suivi de la réalisation concrète de l'étude sur le terrain ou en laboratoire.

  • Activités en cours — liste des activités planifiées avec suivi d'exécution
  • Dates réelles — enregistrement des dates effectives de réalisation
  • Indicateurs de progression par type d'étude
  • Vue sur les phases critiques et leur statut
Pour les études Non-GLP, cette étape est numérotée Étape 4.

Étape 6 — Quality Assurance (GLP uniquement)

Qui : QA Manager (écriture), Facility Manager (lecture + QA Statement), Super Admin

Module complet de suivi qualité :

📌 Inspections QA

Planification et suivi des inspections de qualité. Pour chaque inspection :

  • Type d'inspection (process, facility, study protocol, study report, amendment, data quality)
  • Activité concernée (liée au planning)
  • Date prévue et date réelle de réalisation
  • Checklist d'inspection — formulaire structuré avec réponses Oui/Non/N/A et observations → voir
  • Findings (NC) — non-conformités documentées avec gravité, description, recommandation
  • Résolution des findings avec preuve de correction

📌 Finalisation d'une inspection

Le QA Manager clique sur "Marquer comme terminée" une fois tous les findings documentés. Ce bouton est disponible dès que la date de réalisation est renseignée.

📌 QA Statement

Document récapitulatif généré en PDF une fois toutes les inspections terminées. Il liste les inspections réalisées, les findings, les résolutions et les signatures. Signable électroniquement par le QA Manager, le Study Director et le Facility Manager.

Pour les études Non-GLP, l'étape QA est désactivée (onglet grisé).

Étape 7 — Report Phase

Qui : Facility Manager, Study Director, Project Manager, Super Admin

Dépôt et suivi des documents du rapport final :

  • Upload de documents (brouillon, version finale, amendements)
  • Statut de chaque document (en attente, soumis, approuvé)
  • Date de soumission et commentaires

Cette étape est complète lorsqu'au moins un document de rapport est marqué soumis.

Étape 8 — Archiving Phase

Qui : Archivist, Facility Manager, Super Admin

Clôture officielle de l'étude :

📌 Checklist d'archivage

Liste de vérification à cocher avant de finaliser l'archivage (vérification des documents, des signatures, du dossier physique, etc.).

📌 Documents d'archivage

Upload des documents finaux destinés à l'archive : rapport final, annexes, autorisation d'archivage, etc.

📌 Archiver le projet

Une fois tous les éléments en place, le bouton "Archiver l'étude" verrouille définitivement la fiche. Tous les onglets 1–7 passent en lecture seule. La fiche peut être désarchivée si nécessaire (Facility Manager / Super Admin uniquement).

Pour les études Non-GLP : cette étape est numérotée Étape 6.

Progresser entre les étapes

Chaque étape dispose d'une barre en bas (Phase Complete Bar) qui indique :

  • Le statut de l'étape courante (complète ou non)
  • Un bouton pour marquer l'étape comme complète (ou incomplète)
  • Les prérequis manquants si l'étape n'est pas complémentable

Le bandeau en haut de la fiche affiche la phase courante, le taux d'avancement global, et un bouton "Go to step →" pour naviguer rapidement vers l'onglet correspondant.
