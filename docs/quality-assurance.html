Assurance Qualité (QA)

Module dédié à la gestion des inspections, des constatations et du suivi des non-conformités pour les projets GLP. Accessible via Quality Assurance dans la navigation.

🔐
La phase QA est disponible uniquement pour les projets GLP. Le tableau de bord QA est accessible aux rôles super_admin facility_manager qa_manager.

📊 Tableau de bord QA

Le tableau de bord QA (/qa-dashboard) offre une vue consolidée de toutes les activités qualité :

Constatations non résolues par projet.

🔍

Inspections en attente

Activités critiques

La sous-page Critical Activities (/qa-dashboard/critical-activities) liste toutes les activités marquées comme critiques à travers les projets GLP, avec leur statut d'inspection associé.

🔍 Gestion des inspections

Types d'inspection

TypeDescription
Study InspectionInspection générale de l'étude
Critical Phase InspectionInspection liée à une phase critique identifiée en planification
Data Quality InspectionVérification de la qualité des données
Facility InspectionInspection des installations
Process InspectionInspection des procédés

Planifier une inspection

  1. Dans la phase QA du projet, cliquez sur Planifier une inspection
  2. Sélectionnez le type d'inspection
  3. Choisissez l'activité associée (si Critical Phase)
  4. Saisissez la date planifiée
  5. Assignez un inspecteur
  6. Cliquez sur Enregistrer

Marquer une inspection comme réalisée

Une fois l'inspection effectuée :

  1. Cliquez sur Marquer comme réalisée dans la liste des inspections
  2. Entrez la date réelle de réalisation
  3. L'inspection passe au statut done

Compléter une inspection

Le bouton Compléter finalise l'inspection après résolution de toutes les constatations. L'état peut être basculé (toggle).

📋 Constatations & Non-conformités

Les constatations sont des observations faites lors d'une inspection. Une constatation peut être une non-conformité (NC) ou une conformité.

Ajouter une constatation

  1. Ouvrez l'inspection concernée
  2. Cliquez sur Ajouter une constatation
  3. Renseignez la description
  4. Sélectionnez Non-conformité ou Conformité
  5. Choisissez la sévérité (si NC)
  6. Assignez à un responsable
  7. Enregistrez

super_admin facility_manager qa_manager

Résoudre une non-conformité

Pour chaque NC ouverte :

  1. Cliquer sur Résoudre dans la liste des constatations
  2. Décrire l'action corrective effectuée
  3. Le statut passe de open à complete
  4. Une notification est envoyée aux parties prenantes

Modifier / Supprimer une constatation

  • Modifier : corriger la description ou les informations via le bouton d'édition
  • Supprimer : retirer définitivement la constatation (confirmation requise)
  • Supprimer une action corrective : retirer l'action corrective associée

📄 Déclaration QA (QA Statement)

La déclaration QA est un document formel attestant la conformité de l'étude au regard des exigences GLP.

Créer une déclaration QA

  1. Dans la phase QA du projet, localisez la section QA Statement
  2. Saisissez le numéro de rapport
  3. Entrez la date de signature
  4. Assignez le responsable
  5. Cliquez sur Enregistrer

Signature électronique

La déclaration QA peut être signée électroniquement. Voir → Signatures électroniques.

Impression

Le bouton Imprimer génère un PDF de la déclaration QA.

✅ Checklist d'activités QA

La QA Activities Checklist (référence QA-PR-1-011/05) est une checklist de 20 items spécifique aux projets GLP. Elle pré-remplit automatiquement les dates clés à partir des données du projet.

Accès

  • Depuis la phase QA du projet → bouton Open Checklist
  • Le bouton Télécharger est toujours actif (même sur projet archivé)
  • Le bouton Open Checklist est désactivé si le projet est archivé

Contenu de la checklist

Les 20 items couvrent les jalons GLP : développement du protocole, initiation de l'étude, inspections planifiées et réalisées, rapports, archivage. Les dates sont pré-remplies à partir du projet.

Sauvegarde & Impression

  • Les réponses sont sauvegardées automatiquement via AJAX
  • Bouton Imprimer pour générer un PDF

🏢 Revue QA Facility Manager

Les revues QA Facility Manager (/fm/qa-review) permettent au FM de conduire ses propres audits de conformité.

Créer une revue FM

  1. Cliquez sur Facility Manager dans la navigation
  2. Sélectionnez QA Review Inspections
  3. Cliquez sur Nouvelle revue
  4. Sélectionnez le projet et le modèle de revue
  5. Enregistrez pour créer la revue

Saisir les réponses

Dans la vue détail de la revue :

  • Répondez à chaque question du modèle
  • Ajoutez des éléments personnalisés si nécessaire
  • Sauvegardez les réponses

Compléter & Imprimer

  • Compléter : finalise la revue, statut passe à completed
  • Imprimer : génère un PDF de la revue
  • Supprimer : retire la revue (avec confirmation)
