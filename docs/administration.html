Administration

Gestion des utilisateurs, paramètres globaux de l'application, administration des modèles de checklists et des sections CPIA.

🔒
Les fonctions d'administration sont strictement réservées aux rôles super_admin (toutes fonctions) et facility_manager (paramètres uniquement). Un accès non autorisé retourne une erreur 403.

👥 Gestion des utilisateurs

Accessible via Settings → Gestion utilisateurs ou /admin/users.

super_admin uniquement

Liste des utilisateurs

La page affiche tous les utilisateurs enregistrés avec :

  • Nom et e-mail
  • Rôle actuel (affiché sous forme de badge coloré)
  • Date de création du compte

Modifier le rôle d'un utilisateur

  1. Repérez l'utilisateur dans la liste
  2. Cliquez sur le menu déroulant de son rôle
  3. Sélectionnez le nouveau rôle parmi les 7 disponibles
  4. La modification est sauvegardée automatiquement
⚠️
Changer le rôle d'un utilisateur modifie immédiatement ses accès. Soyez prudent lors de la modification du rôle d'un utilisateur actif.

Rôles disponibles

super_admin facility_manager qa_manager study_director project_manager archivist read_only

🔧 Paramètres de l'application

Accessible via Settings → Paramètres ou /settings.

super_admin facility_manager

La page de paramètres permet de configurer les options globales de l'application :

  • Configuration des paramètres de messagerie (e-mail sender, etc.)
  • Paramètres de l'organisation
  • Options d'affichage globales

Sauvegarder les paramètres

  1. Modifiez les valeurs souhaitées dans le formulaire
  2. Cliquez sur Enregistrer les paramètres
  3. Les nouveaux paramètres s'appliquent immédiatement

📋 Administration des checklists

Accessible via Settings → Questions Checklists ou /admin/checklists.

super_admin uniquement

Vue d'ensemble des modèles

La page liste les 6 modèles de checklists disponibles. Cliquer sur un modèle ouvre son éditeur.

Gérer les questions

Pour chaque modèle, vous pouvez gérer les questions par section :

ActionDescription
Ajouter une question Crée une nouvelle question dans une section donnée
Modifier une question Met à jour le texte ou le type de réponse
Dupliquer une question Crée une copie de la question dans la même section
Activer / Désactiver Bascule la visibilité de la question dans les checklists (toggle)
Supprimer Supprime définitivement la question (avec confirmation)
⚠️
La suppression d'une question est irréversible et peut affecter les inspections existantes qui utilisent cette question. Préférez la désactivation.

Ajouter une question

  1. Ouvrez le modèle souhaité
  2. Naviguez vers la section concernée
  3. Cliquez sur + Ajouter une question
  4. Saisissez le texte de la question
  5. Sélectionnez le type de réponse attendu
  6. Enregistrez

⚠️ Administration des sections CPIA

Accessible via Settings → CPIA Sections ou /admin/cpia.

super_admin uniquement

Vue d'ensemble

La page liste toutes les sections CPIA. Cliquer sur une section ouvre son éditeur d'items.

Gérer les items CPIA

ActionDescription
Ajouter un item Crée un nouvel item dans la section (POST admin.cpia.items.store)
Modifier un item Met à jour le texte ou le niveau de risque de l'item
Dupliquer un item Crée une copie de l'item dans la même section
Activer / Désactiver Bascule la visibilité de l'item dans les formulaires CPIA
Supprimer Supprime définitivement l'item
💡
Pour retirer un item sans perdre l'historique des évaluations existantes, utilisez Désactiver plutôt que Supprimer. Les items désactivés ne s'affichent plus dans les nouveaux formulaires CPIA.
