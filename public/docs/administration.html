⚙️ Administration

Le panneau d'administration est accessible uniquement aux utilisateurs avec le rôle Super Admin. Il permet de gérer les comptes utilisateurs, les contenus structurants du système (sections CPIA, templates de checklists, personnel clé) et les paramètres de l'application.

Accéder au panneau d'administration

Cliquez sur votre nom en haut à droite de la navigation, puis sur "Administration" dans le menu déroulant. Ce lien n'est visible que pour les Super Admins.

Toutes les actions dans le panneau admin sont irréversibles (suppression de compte, désactivation d'un item). Procéder avec précaution.

👤 Gestion des utilisateurs

Accessible via Admin → Utilisateurs. Cette vue liste tous les comptes enregistrés sur la plateforme.

Créer un compte

La création de compte se fait via la page de Register (lien sur la page de connexion ou /register). Après inscription, le Super Admin attribue un rôle à l'utilisateur.

Modifier le rôle d'un utilisateur

Dans la liste des utilisateurs, chaque ligne affiche le nom, l'email et le rôle actuel. Un menu déroulant permet de changer le rôle immédiatement. Les rôles disponibles sont :

RôleDescription
Super AdminAccès total à toutes les fonctionnalités et au panneau admin
Facility ManagerGestion complète des projets, QA Statement, archivage, paramètres
QA ManagerInspections, checklists, findings, CPIA, QA Statement
Study DirectorCréation d'études, protocol, planning, rapport
Project ManagerProtocol, planning, rapport — pas de QA ni archivage
ArchivistÉtape archivage uniquement
Read OnlyLecture seule sur toutes les sections
Le changement de rôle est immédiat — la prochaine action de l'utilisateur applique déjà les nouvelles permissions.

⚠️ Gestion des sections CPIA

Accessible via Admin → Sections CPIA. La CPIA (Critical Phase Impact Assessment) est structurée en sections (A, B, C…), chacune contenant des items évaluables. Ce module permet de personnaliser ce contenu.

Sections

Les sections sont affichées avec leur lettre, leur titre et le nombre d'items actifs. Elles sont en lecture seule dans l'interface actuelle (les titres peuvent être modifiés en base si besoin).

Items d'une section

Cliquez sur une section pour accéder à sa liste d'items. Pour chaque item vous pouvez :

  • Modifier le texte — édition inline avec sauvegarde immédiate
  • Activer / Désactiver — un item désactivé n'apparaît plus dans la CPIA (mais son historique est conservé)
  • Dupliquer — crée une copie de l'item dans la même section
  • Supprimer — suppression définitive (impossible si des réponses existent)
  • Ajouter un item — formulaire en bas de la liste
La numérotation des items est automatique. Modifier l'ordre ou supprimer un item peut affecter des évaluations déjà en cours.

✅ Gestion des checklists d'inspection

Accessible via Admin → Checklists. Les checklists sont organisées par template (ex. : Facility Inspection, Process Inspection, Study Protocol…). Chaque template contient des sections, elles-mêmes composées de questions.

Templates disponibles

  • Facility Inspection Checklist
  • Process Inspection Checklist
  • Study Protocol Inspection Checklist
  • Study Report Inspection Checklist
  • Amendment Inspection Checklist
  • Data Quality Inspection Checklist

Gérer les questions d'une section

Cliquez sur un template pour voir ses sections et questions. Chaque question peut être :

  • Modifiée — texte éditable inline
  • Activée / Désactivée — contrôle la visibilité dans le formulaire d'inspection
  • Dupliquée — copie dans la même section
  • Supprimée — uniquement si aucune réponse n'existe pour cette question
  • Ajoutée — nouveau formulaire en bas de section
Les modifications de questions affectent uniquement les nouvelles inspections. Les inspections déjà créées conservent une copie instantanée (snapshot) des questions au moment de leur création.

🧑‍🔬 Personnel Clé de la Facilité (Key Facility Personnel)

Le personnel clé est le référentiel des agents AIRID qui peuvent être assignés aux études (comme Study Director, Project Manager, Key Personnel). Ce référentiel est géré par le Facility Manager ou le Super Admin.

Pour chaque agent, les informations disponibles sont :

  • Titre, prénom, nom
  • Qualification (titre académique ou professionnel)
  • Rôle principal (Study Director, Lab Technician, etc.)
  • Email professionnel
  • Compte utilisateur associé (pour les notifications)
Le lien entre un agent du personnel et un compte utilisateur (user_id) est essentiel pour que les notifications in-app fonctionnent correctement. Si un agent n'a pas de compte, il ne recevra pas de notification.

🔧 Paramètres de l'application

Accessible via Admin → Paramètres (Facility Manager + Super Admin). Permet de configurer des valeurs globales utilisées dans l'application :

  • Nom de la facilité
  • Numéro d'accréditation SANAS
  • Logo et informations institutionnelles
  • Emails de notification par défaut

Ces paramètres alimentent les en-têtes des documents PDF et des rapports générés.

🔍 QA Review (Facility Manager)

Accessible via FM → QA Review. Ce module permet au Facility Manager de réaliser ses propres revues qualité indépendamment des inspections QA Manager. Une QA Review contient :

  • Date et objet de la revue
  • Réponses structurées par sections (similaire aux checklists)
  • Statut (en cours / complétée)
  • Impression PDF de la revue

🔐 Bonnes pratiques de sécurité

  • Ne partagez jamais votre mot de passe — chaque action est tracée avec l'identifiant de l'utilisateur connecté
  • Déconnectez-vous après chaque session sur un poste partagé
  • Les mots de passe sont stockés chiffrés (bcrypt) — en cas d'oubli, utilisez "Mot de passe oublié ?" sur la page de connexion
  • Le Super Admin peut réinitialiser un rôle mais pas un mot de passe — l'utilisateur doit le faire lui-même via le lien de réinitialisation
  • Tous les formulaires sont protégés par un jeton CSRF — les liens copiés depuis d'autres onglets peuvent expirer
