CPIA — Critical Phase Impact Assessment

Le CPIA est une évaluation formelle permettant d'identifier les phases critiques d'une étude GLP et d'en calculer le niveau de risque global.

🔐
Le CPIA est accessible depuis la Phase 4 (Planning) d'un projet GLP. Rôles autorisés pour remplir et compléter : super_admin facility_manager study_director project_manager.

📖 Qu'est-ce que le CPIA ?

Le Critical Phase Impact Assessment (CPIA) est un outil d'évaluation qui permet de :

  • Identifier les phases de l'étude à fort impact sur la qualité des données
  • Qualifier chaque phase comme "critique" ou "non-critique"
  • Calculer automatiquement le niveau de risque global de l'étude
  • Documenter formellement les décisions pour les auditeurs GLP

🚀 Accéder au CPIA

  1. Ouvrez le projet GLP concerné
  2. Naviguez vers l'onglet Phase 4 — Planning
  3. Cliquez sur le bouton CPIA dans la section Planning
  4. Vous êtes redirigé vers /project/{project_id}/cpia

✏️ Remplir le CPIA

Structure de l'évaluation

Le formulaire CPIA est organisé en sections, chacune contenant plusieurs items d'évaluation. Pour chaque item :

  • Cochez Critique si la phase est identifiée comme critique
  • Ajoutez un commentaire optionnel pour justifier votre choix
  • Les items peuvent être actifs ou inactifs (gérés par l'administrateur)

Sauvegarder les réponses

  1. Remplissez les items de chaque section
  2. Cliquez sur Sauvegarder
  3. Le système calcule automatiquement le niveau de risque
  4. L'évaluation passe en statut draft (modifiable)

super_admin facility_manager study_director project_manager

📊 Calcul du niveau de risque

Le niveau de risque est calculé automatiquement lors de la sauvegarde en fonction du nombre de phases critiques identifiées :

Nombre de phases critiquesNiveau de risque
0⬤ Faible
1 — 3⬤ Modéré
4 et plus⬤ Élevé

Le résultat s'affiche en haut du formulaire et dans le résumé du projet.

✅ Compléter le CPIA

Une fois l'évaluation finalisée :

  1. Vérifiez qu'au moins une phase critique a été identifiée (requis pour la complétion)
  2. Cliquez sur Compléter le CPIA
  3. Le statut passe de draft à completed
  4. Le formulaire passe en lecture seule
⚠️
Si aucune phase critique n'est cochée, le système refusera la complétion et affichera un message d'erreur.

super_admin facility_manager study_director project_manager

↩️ Remettre en brouillon

Si vous devez modifier un CPIA déjà complété :

  1. Cliquez sur Remettre en brouillon (Revert to Draft)
  2. Le statut repasse à draft
  3. Le formulaire redevient éditable

super_admin facility_manager study_director project_manager

🖨️ Imprimer le CPIA

Le bouton Imprimer génère un PDF formaté du CPIA contenant :

  • En-tête avec code et titre du projet
  • Toutes les sections avec les items et réponses
  • Commentaires saisis
  • Résumé : nombre de phases critiques et niveau de risque
  • Date de complétion et signataire

super_admin facility_manager study_director project_manager

⚙️ Administration des sections CPIA

Le super_admin peut gérer les sections et items CPIA depuis /admin/cpia. Voir → Administration CPIA.
