Créer & Gérer un projet

L'assistant de gestion de projet (Study Management Wizard) guide l'utilisateur à travers jusqu'à 8 phases pour les projets GLP, et 6 phases pour les projets non-GLP.

🔐
Qui peut créer un projet ? Uniquement les rôles super_admin et facility_manager.

➕ Créer un nouveau projet

Depuis le tableau de bord, cliquez sur Study Management dans la navigation, puis sur le bouton Nouveau projet. Une boîte de dialogue s'ouvre :

  • Entrez le code du projet (ex. : BEN-2025-001)
  • Entrez le titre du projet
  • Sélectionnez si le projet est GLP ou non (détermine le nombre de phases)
  • Cliquez sur Créer pour initialiser le projet

Le projet est créé et vous êtes redirigé vers l'assistant de gestion (wizard).

📋 Vue d'ensemble des phases

Le wizard affiche toutes les phases sous forme d'onglets. Chaque phase indique son état d'avancement. Les phases marquées GLP uniquement ne s'affichent que pour les projets GLP.

1

Study Creation — Création de l'étude

Informations de base du projet, nomination du Study Director, formulaires d'appointment/remplacement, personnel clé.

2

Protocol Details — Détails du protocole

Code protocole, types d'étude, types de produits évalués, tests de laboratoire concernés.

3

Protocol Development — Développement du protocole

5 niveaux de documents à compléter : Protocol Draft (L1), Final Protocol (L2), Study Plan (L3), Monitoring Plan (L4), Amendment/Deviation (L5 — optionnel).

4

Planning — Planification

Organisation de la réunion d'initiation de l'étude, identification des phases critiques, évaluation CPIA.

GLP uniquement
5

Experimental — Phase expérimentale

Planification et exécution des activités, suivi des statuts (pending / in_progress / completed / delayed), historique d'exécution.

6

Quality Assurance — Assurance Qualité

Planification des inspections QA, suivi des constatations et non-conformités, déclaration QA finale, checklist d'activités QA (20 items GLP).

GLP uniquement
7

Reporting — Rapport

Soumission des rapports (draft report, final report), suivi des statuts de publication.

8

Archiving — Archivage

Checklist d'archivage (18 items), soumission des documents d'archivage, validation finale et archivage du projet.

Phase 1 — Study Creation

Informations de base

  • Modification du code, titre, statut et type GLP/non-GLP du projet
  • Dates de début et fin effectives
  • Statut du projet (not_started, in progress, suspended, completed)

super_admin facility_manager

Nomination du Study Director

Le formulaire de nomination enregistre :

  • Le Study Director sélectionné
  • La date de nomination
  • La signature électronique

En cas de remplacement, un second formulaire est disponible pour documenter le changement.

super_admin facility_manager

Personnel clé (Key Personnel)

  • Ajout de membres de l'équipe projet via le modal de sélection
  • Retrait d'un membre existant
  • Chaque membre ajouté reçoit une notification automatique

super_admin facility_manager

Phase 2 — Protocol Details

  • Code protocole : identifiant unique du protocole
  • Types d'étude : In vitro, In vivo, Observational, etc. (sélection multiple)
  • Types de produits évalués : LLIN, IRS, Repellent, etc. (sélection multiple)
  • Tests de laboratoire : sélection des tests concernés par le projet

super_admin facility_manager study_director project_manager

Phase 3 — Protocol Development

Cette phase est structurée en 5 niveaux de documents. Les niveaux 1 à 4 sont obligatoires, le niveau 5 est optionnel.

NiveauDocumentObligatoire
L1Protocol Draft✓
L2Final Protocol✓
L3Study Plan✓
L4Monitoring Plan✓
L5Amendment / DeviationOptionnel

Pour chaque niveau, vous pouvez :

  • Générer les activités de développement
  • Marquer l'activité comme complète avec date de réalisation
  • Uploader les documents associés
  • Supprimer un document ou une entrée de document

super_admin facility_manager study_director project_manager

Phase 4 — Planning GLP uniquement

Réunion d'initiation

Planifiez la réunion de démarrage de l'étude (Study Initiation Meeting) :

  • Date et heure prévues
  • Participants
  • Lieu ou lien de réunion

Une notification par e-mail est envoyée aux participants à la création de la réunion.

Phases critiques

  • Marquer/démarquer les activités comme "phase critique"
  • Planifier une réunion spécifique pour les phases critiques
  • Les phases critiques déclenchent les inspections QA de type "Critical Phase Inspection"

CPIA

Accès direct à l'évaluation CPIA depuis cette phase. → Voir la documentation CPIA

super_admin facility_manager study_director project_manager

Phase 5 — Experimental

Gestion des activités

Chaque activité dispose des informations suivantes :

  • Nom de l'activité
  • Responsable assigné
  • Date estimée
  • Statut : pending / in_progress / completed / delayed
  • Date réelle de réalisation

Actions sur les activités

  • Ajouter une activité via le formulaire modal
  • Exécuter une activité : marque comme réalisée avec date et utilisateur
  • Réinitialiser : revenir au statut précédent
  • Supprimer une activité (avec confirmation)
  • Historique d'exécution : journal complet des actions effectuées sur l'activité

super_admin facility_manager study_director project_manager

Phase 6 — Quality Assurance GLP uniquement

Voir la documentation détaillée : → Assurance Qualité

Cette phase inclut :

  • Planification et suivi des inspections QA
  • Enregistrement des constatations (non-conformités)
  • Résolution des NC et actions correctives
  • Déclaration QA finale (QA Statement)
  • Checklist d'activités QA (20 items GLP, QA-PR-1-011/05)

super_admin facility_manager qa_manager

Phase 7 — Reporting

  • Soumettre un Draft Report avec titre et date de soumission
  • Soumettre un Final Report après validation
  • Mettre à jour le statut : submitted → published
  • Modifier ou supprimer un document de rapport

super_admin facility_manager

Phase 8 — Archiving

Checklist d'archivage

Une checklist de 18 items doit être complétée avant l'archivage :

  • Vérification de la présence de tous les documents obligatoires
  • Chaque item peut être coché comme complété
  • Sauvegarde automatique de la progression

Documents d'archivage

  • Soumettre les documents physiques archivés avec type et date
  • Supprimer un document si nécessaire

Archivage du projet

  1. Toutes les phases doivent être complètes
  2. Cliquer sur Archiver le projet
  3. Le projet passe en lecture seule (archived_at est défini)
  4. Le statut change à Archived
⚠️
Un projet archivé désactive tous les boutons de modification. Le bouton "Désarchiver" reste disponible pour super_admin et facility_manager si nécessaire.

super_admin facility_manager archivist

✅ Complétion des phases

Chaque phase peut être marquée comme complète de deux façons :

🤖

Automatique

Basé sur des critères définis (ex. : toutes les activités complètes, tous les documents soumis).

✋

Manuelle

Via le bouton "Compléter la phase" pour forcer le statut même si les critères ne sont pas tous remplis.

🔍 Vue détaillée du projet (Project Overview)

Accessible depuis la carte du projet sur le tableau de bord, cette vue récapitule :

  • Toutes les phases et leur statut de complétion
  • Liste des inspections QA et leurs constatations
  • Documents téléchargeables (selon les droits)
  • Score global d'avancement calculé
  • Équipe du projet
